### What was built

**`backend/src/lib/manifestEngine.js`** — `computeManifest()` step 3 replaced

Before: single `INNER JOIN schedules → content ON content_id` — silently dropped every playlist-based schedule (content_id IS NULL).

After: two parallel queries merged:

- **Query A** (`contentSchedRes`): schedules with `content_id IS NOT NULL`, INNER JOIN content, filters `c.expires_at`. Identical semantics to old query plus explicit NULL guard.
- **Query B** (`playlistSchedRes`): schedules with `playlist_id IS NOT NULL`, JOIN `named_playlists`, `CROSS JOIN LATERAL jsonb_array_elements(np.items)`, JOIN content on `(item->>'content_id')::uuid`. Expands each playlist item into a separate schedule row with per-item duration from `duration_seconds`.
- Both queries share the same `TARGET_FILTER` string (screen/group/venue/global fallback).
- Merged: `[...contentSchedRes.rows, ...playlistSchedRes.rows].sort((a, b) => b.priority - a.priority)`
- All downstream logic (`scheduleActive`, `toItems`, fallback, legacy bridge, system fallback) unchanged — operates on `allSchedRows` instead of `schedRes.rows`.

**`backend/src/routes/resolve.js`** — new file

- `GET /resolve/:screen_id` — adapter for player-runtime `PlaylistPoller`
- Calls `getManifest(screen_id)` (cached, max 5s stale)
- Transforms manifest into `ResolvedPlaylist` shape:
  - `playlist`: array of `{ content_id, duration_ms, weight, source, sponsored }`
  - `playlist_checksum`: manifest's FNV-1a checksum (8 hex chars)
  - `resolution_level`: 1 if items present, 0 if empty
  - `is_fallback`: true only if all items have `source === 'fallback'`
  - `_meta`: `{ correlation_id, at_utc_ms, venue_id }`
- No `screenAuth.requireScreenToken` — player-runtime dev/sim mode has no token; `SCREEN_AUTH_ENFORCE` defaults false

**`backend/src/index.js`**

- Added `const resolveRouter = require('./routes/resolve')`
- Mounted: `app.use('/resolve', rateLimit(120, 60_000), resolveRouter)` — after `/manifest`, before `/content`

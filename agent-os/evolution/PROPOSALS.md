# Agent OS — Evolution Proposals

This file tracks proposed improvements to Agent OS itself. Proposals come from projects using Agent OS — specifically from the Governance role observing patterns that likely generalise beyond one project.

**Lifecycle**: Observation → Draft → In Review → Accepted / Rejected → Implemented

See `workflows/periodic-review.md` for when and how to generate proposals.

---

## How to Submit a Proposal

Any project's Governance role can submit a proposal. Fill in the template below and append it to this file under the correct status section.

```markdown
### P-NNN — [Short title]

**Submitted by**: [Project name or description] — [Date]
**Status**: Draft

**Observation**:
What was observed in the project? Be specific: what happened, how many times, what was the cost?

**Root cause**:
Why does the current Agent OS design cause or fail to prevent this?

**Proposed change**:
What specifically should change? Which file(s)? What text?

**Expected benefit**:
How would this change reduce interruptions or improve agent performance?

**Generalises because**:
Why would this improvement help other projects, not just this one?

**Tradeoffs / risks**:
What could go wrong? What might this break or complicate?
```
